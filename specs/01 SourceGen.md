# Serde.FS.SourceGen — Implementation Spec

## Overview

`Serde.FS.SourceGen` is an MSBuild-time code generator that runs during compilation of projects referencing `Serde.FS.STJ`. It uses the **FSharp.Compiler.Service** (FCS) to parse F# source files, find types annotated with `[<Serde>]`, `[<SerdeSerialize>]`, or `[<SerdeDeserialize>]`, and emit companion F# source files containing optimized, reflection-free serialization/deserialization code for the System.Text.Json backend.

It is **not** a Roslyn `ISourceGenerator`. F# does not support Roslyn source generators. Instead, it is an MSBuild task that shells out to FCS for parsing, then writes generated `.fs` files that are included in compilation.

## Existing Core Types (defined in Serde.FS)

These are the types the generator must understand and target. Do NOT modify these files.

### Attributes (`Serde.FS/Attributes.fs`)
```fsharp
namespace Serde.FS

[<AttributeUsage(AttributeTargets.Class ||| AttributeTargets.Struct, AllowMultiple = false)>]
type SerdeAttribute() = inherit Attribute()

[<AttributeUsage(AttributeTargets.Class ||| AttributeTargets.Struct, AllowMultiple = false)>]
type SerdeSerializeAttribute() = inherit Attribute()

[<AttributeUsage(AttributeTargets.Class ||| AttributeTargets.Struct, AllowMultiple = false)>]
type SerdeDeserializeAttribute() = inherit Attribute()
```

### Backend interface (`Serde.FS/ISerdeBackend.fs`)
```fsharp
type ISerdeBackend =
    abstract Serialize : 'T * ISerdeOptions option -> string
    abstract Deserialize : string * ISerdeOptions option -> 'T
```

### STJ Backend (`Serde.FS.STJ/JsonBackend.fs`)
```fsharp
type JsonBackend() =
    interface ISerdeBackend with
        member _.Serialize(value, options) =
            let opts =
                match options with
                | Some (:? JsonBackendOptions as o) -> o.JsonOptions
                | _ -> JsonSerializerOptions()
            JsonSerializer.Serialize(value, opts)

        member _.Deserialize(json, options) =
            let opts =
                match options with
                | Some (:? JsonBackendOptions as o) -> o.JsonOptions
                | _ -> JsonSerializerOptions()
            JsonSerializer.Deserialize<'T>(json, opts)
```

The current `JsonBackend` uses `JsonSerializer.Serialize/Deserialize` which relies on runtime reflection. The source generator's job is to produce `JsonTypeInfo<'T>` metadata via `System.Text.Json.Serialization.Metadata` so that STJ can serialize without reflection.

## What the Generator Must Do

### Step 1: Parse

Use FCS to parse all F# source files in the consuming project. For each file:

1. Call `FSharpChecker.ParseFile` to obtain an untyped AST (`ParsedInput`).
2. Walk the AST looking for type definitions decorated with one of:
   - `Serde` / `SerdeAttribute`
   - `SerdeSerialize` / `SerdeSerializeAttribute`
   - `SerdeDeserialize` / `SerdeDeserializeAttribute`
3. For each matched type, extract:
   - Fully qualified type name (namespace + type name)
   - Kind: record, single-case DU wrapper, or standard DU
   - Fields: name, F# type syntax (as written in source)

**Important:** Use *untyped* AST parsing only (`parseFileResults.ParseTree`). Do NOT run a full typecheck — it is too slow for build-time use and requires resolved references. Attribute detection should match on the short name (e.g., `Serde`) and the long name (e.g., `SerdeAttribute`).

### Step 2: Emit

For each discovered type, generate an F# source file that provides a `JsonTypeInfo<'T>` so System.Text.Json can serialize/deserialize without reflection.

#### Target: STJ JsonTypeInfo via `JsonMetadataServices`

The generated code should produce a `System.Text.Json.Serialization.Metadata.JsonTypeInfo<'T>` for each annotated type. This is the same approach that the C# `[JsonSerializable]` source generator uses internally.

#### Example

Given this user code:
```fsharp
namespace MyApp

open Serde.FS

[<Serde>]
type Person = { FName: string; LName: string; Age: int }
```

The generator should emit a file like `Person.serde.g.fs`:
```fsharp
// <auto-generated />
namespace MyApp

open System
open System.Text.Json
open System.Text.Json.Serialization.Metadata

[<AutoOpen>]
module internal PersonSerdeTypeInfo =

    let personJsonTypeInfo (options: JsonSerializerOptions) : JsonTypeInfo<Person> =
        let info = JsonMetadataServices.CreateObjectInfo<Person>(
            options,
            JsonObjectInfoValues<Person>(
                ObjectCreator = (fun () -> Unchecked.defaultof<Person>),
                SerializeHandler = (fun writer value ->
                    writer.WriteStartObject()
                    writer.WriteString("FName", value.FName)
                    writer.WriteString("LName", value.LName)
                    writer.WriteNumber("Age", value.Age)
                    writer.WriteEndObject()
                )
            )
        )
        info
```

> **Note:** The exact shape of the generated code will evolve. The above is a guiding example, not a rigid template. Focus on producing code that:
> 1. Compiles as valid F#
> 2. Creates a `JsonTypeInfo<'T>` without runtime reflection
> 3. Handles all fields of the record

### Supported Type Mappings (Phase 1)

Map F# types to `Utf8JsonWriter` calls:

| F# type | Writer method |
|---------|--------------|
| `string` | `WriteString` |
| `int` / `int32` | `WriteNumber` |
| `int64` | `WriteNumber` |
| `float` / `double` | `WriteNumber` |
| `decimal` | `WriteNumber` |
| `bool` | `WriteBoolean` |
| `byte[]` | `WriteBase64String` |
| `DateTime` | `WriteString` (ISO 8601) |
| `Guid` | `WriteString` |
| `'T option` | Write value or `WriteNullValue` |

Nested records that are also `[<Serde>]` should delegate to their own generated `JsonTypeInfo`.

### Phase 2 (future, do not implement yet)

- Discriminated unions
- Collection types (`list<'T>`, `array<'T>`, `Map<K,V>`)
- Custom field naming (`[<SerdeRename "first_name">]`)
- Deserialization with `Utf8JsonReader`

## Step 3: Integrate via MSBuild

The generator must hook into the build as an MSBuild task so that generated files are included before the main compilation pass.

### Approach: Custom MSBuild Task

1. Create a class inheriting `Microsoft.Build.Utilities.Task` (add a PackageReference to `Microsoft.Build.Utilities.Core`).
2. The task receives the list of `Compile` items from the consuming project.
3. It runs the FCS parse + emit pipeline.
4. It writes generated `.fs` files to `$(IntermediateOutputPath)/generated/` (i.e., `obj/Debug/net8.0/generated/`).
5. It returns the generated file paths as an output, which MSBuild injects into the `Compile` item list.

### .props / .targets files

Ship a `.targets` file that wires up the task. This file should be included in the NuGet package under `build/` or `buildTransitive/` so consumers get it automatically.

`Serde.FS.STJ.targets` (shipped in the STJ NuGet package):
```xml
<Project>
  <UsingTask TaskName="Serde.FS.SourceGen.SerdeGeneratorTask"
             AssemblyFile="$(MSBuildThisFileDirectory)../analyzers/dotnet/fs/Serde.FS.SourceGen.dll" />

  <Target Name="SerdeGenerate" BeforeTargets="CoreCompile">
    <SerdeGeneratorTask SourceFiles="@(Compile)" OutputDir="$(IntermediateOutputPath)serde-generated/" />
    <!-- Add generated files to Compile, inserted before user code -->
    <ItemGroup>
      <Compile Include="$(IntermediateOutputPath)serde-generated/**/*.fs"
               Condition="Exists('$(IntermediateOutputPath)serde-generated/')" />
    </ItemGroup>
  </Target>
</Project>
```

> **Note:** The generated files must appear AFTER the types they extend in F#'s compilation order. Since they go into a separate directory and are appended to `Compile`, they will compile after user source files. This is correct — the generated modules reference the user-defined types.

## File / Module Structure

Suggested layout for the SourceGen project:

```
src/Serde.FS.SourceGen/
  Serde.FS.SourceGen.fsproj
  AstParser.fs          — FCS AST walking, attribute detection, type extraction
  TypeModel.fs          — Internal model: SerdeTypeInfo, FieldInfo, etc.
  CodeEmitter.fs        — F# code generation (string building)
  SerdeGeneratorTask.fs — MSBuild Task entry point
```

Compilation order in fsproj:
```xml
<ItemGroup>
  <Compile Include="TypeModel.fs" />
  <Compile Include="AstParser.fs" />
  <Compile Include="CodeEmitter.fs" />
  <Compile Include="SerdeGeneratorTask.fs" />
</ItemGroup>
```

## AstParser

```fsharp
namespace Serde.FS.SourceGen

module AstParser =
    /// Parse an F# source file and return all Serde-annotated types found.
    val parseFile : filePath: string -> SerdeTypeInfo list
```

Implementation notes:
- Create an `FSharpChecker` instance (use `FSharpChecker.Create()`).
- Call `checker.ParseFile(filePath, sourceText, parsingOptions)`.
- Walk `parseFileResults.ParseTree` recursively.
- Match on `SynTypeDefn` nodes.
- Check `SynAttributes` for Serde attribute names.
- Extract fields from `SynTypeDefnRepr.Simple(SynTypeDefnSimpleRepr.Record(...))`.

## CodeEmitter

```fsharp
namespace Serde.FS.SourceGen

module CodeEmitter =

```

The emitted string must be a complete, compilable F# source file.

## SerdeGeneratorTask

```fsharp
namespace Serde.FS.SourceGen

open Microsoft.Build.Utilities
open Microsoft.Build.Framework

type SerdeGeneratorTask() =
    inherit Task()

    [<Required>]
    member val SourceFiles : ITaskItem array = [||] with get, set

    member val OutputDir : string = "" with get, set

    override this.Execute() =
        // 1. Parse each source file via AstParser.parseFile
        // 2. For each SerdeTypeInfo, call CodeEmitter.emit
        // 3. Write to OutputDir/{TypeName}.serde.g.fs
        // 4. Return true on success
        true
```

## Testing Strategy

Add tests in `Serde.FS.STJ.Tests` (or a new `Serde.FS.SourceGen.Tests` project) that:

1. **AstParser unit tests:** Feed small F# source strings to `parseFile` and assert the returned `SerdeTypeInfo` list.
2. **CodeEmitter unit tests:** Feed a `SerdeTypeInfo` to `emit` and assert the output compiles (or snapshot-test the string).
3. **Integration test:** Create a temp F# file with `[<Serde>] type Foo = { X: int }`, run the full pipeline, compile the output, and verify it produces a working `JsonTypeInfo`.

## Constraints

- **No Roslyn.** This is pure FCS-based parsing. Do not reference `Microsoft.CodeAnalysis`.
- **Untyped parse only.** Do not run FCS type-checking. It is too slow and requires full reference resolution. Attribute matching is by syntax name only.
- **Generated code must be valid F#.** Test that emitted code compiles.
- **No modification of existing files.** The generator only adds new files; it never edits user source.
- **Deterministic output.** Same input must produce byte-identical output to avoid unnecessary rebuilds.
- **Phase 1 scope: F# records only.** DUs, classes, and collections are out of scope.
